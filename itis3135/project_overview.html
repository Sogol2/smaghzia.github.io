Project Overview

Project Name: Student Wellness & Fitness Site

Interactive Web Application Requirements & Design Document

Project Overview

  1. Purpose (What the application is & why): A beginner-friendly wellness website led by a UNC Charlotte student to share short weekly challenges, simple workout routines (5–15 minutes), campus fitness resources, and meet-up info. The goal is to make fitness approachable for busy students and encourage participation through small, consistent actions.
  2. Intended Users: UNC Charlotte students (beginner to intermediate), friends/classmates interested in quick routines.
  3. Overview of the Website Content: Weekly challenges with checklists and progress badges; events/meet-ups with RSVP; a resource hub for campus facilities/classes; and a Contact/Join form with basic validation.

Wireframe & Site Map

Page Design Details

Home (index.html)

  1. Purpose: Welcome users, explain the initiative, and highlight the current week’s challenge.
  2. Audience: All visitors (students, friends/classmates).
  3. Content: Hero with “Join this week’s challenge”; quick links to Challenges, Resources, Events, Contact.
  4. Enter Data? (Y/N): No.
  5. Validation? (Y/N): No.
  6. Buttons/Links/Dropdowns: CTA buttons to Challenges and Events.
  7. Actions: Smooth-scroll; navigate to Challenges.
  8. Notes: Keep lightweight and accessible.

Challenges (challenges.html)

  1. Purpose: Provide weekly fitness challenges with simple steps and track progress.
  2. Audience: Students completing routines.
  3. Content: jQuery-UI Accordion or cards per week; checklist items; estimated time; tips.
  4. Enter Data? (Y/N): Yes (checkboxes).
  5. Validation? (Y/N): Yes—require all required steps before “Week Complete.”
  6. Buttons/Links/Dropdowns: “Mark Week Complete” button; optional Beginner/Intermediate tabs.
  7. Actions: Persist progress to localStorage; show badge or small confetti on completion.
  8. Notes: Use ARIA roles and keyboard-friendly focus states.

Resources (resources.html)

  1. Purpose: Centralize campus fitness info (UREC classes, track hours, nearby trails, links).
  2. Audience: Anyone needing quick access to resources.
  3. Content: Resource cards with descriptions and external links; optional filter/search input.
  4. Enter Data? (Y/N): Optional (search/filter text).
  5. Validation? (Y/N): No.
  6. Buttons/Links/Dropdowns: External links open new tab.
  7. Actions: Client-side filter; render items via AJAX from data/resources.json.
  8. Notes: Use descriptive link text; rel="noopener" for security.

Events (events.html)

  1. Purpose: List upcoming meet-ups/workouts and allow simple RSVP.
  2. Audience: Students who plan to attend.
  3. Content: Event cards/table with date, time, location, brief details; datepicker to browse.
  4. Enter Data? (Y/N): Yes (RSVP name/email) — optional if you prefer external signup.
  5. Validation? (Y/N): Yes (required fields and email pattern).
  6. Buttons/Links/Dropdowns: “Add to Calendar” link; RSVP submit.
  7. Actions: AJAX fetch from data/events.json; client-side confirmation toast.
  8. Notes: Graceful empty-state if no events.

Contact / Join (contact.html)

  1. Purpose: Let students ask questions or express interest in joining challenges/meet-ups.
  2. Audience: Students.
  3. Content: Form (name, UNCC email, interest, message).
  4. Enter Data? (Y/N): Yes.
  5. Validation? (Y/N): Yes (required fields, email pattern, min/max length).
  6. Buttons/Links/Dropdowns: Inputs, select, submit.
  7. Actions: JS validate → send via mail handler (Formspree/EmailJS) or mock POST; show success/failure notice.
  8. Notes: No PII is stored on the site; use client-side only.

Dynamic Functionality (JavaScript / jQuery / AJAX)

  1. Challenge Tracker (Challenges): Checkboxes persist to localStorage; progress badge updates in real time; optional confetti on completion.
  2. jQuery-UI Widgets (Challenges & Events): Accordion for weekly challenges; Datepicker to browse events.
  3. Events via AJAX (Events): Load events from data/events.json; render cards/table; empty-state handling.
  4. Resources via AJAX (Resources): Render resource cards from data/resources.json so updates don’t require HTML edits.
  5. Form Validation (Contact): Custom JS validation with helpful messages and focus management for accessibility.
  6. Shared Header/Footer (All pages): Inject components/header.html and components/footer.html for consistent navigation and validator links.
  7. Cardio exercises at home (reference for designing short cardio challenges)
  8. Fitbod — exercises library (reference for selecting one movements and cues)
