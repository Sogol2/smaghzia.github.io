Project Overview / C++ Basics Learning Site

Interactive Web Application Requirements & Design Document

Project Overview

  1. Purpose (What the application is & why): A beginner-friendly website that teaches the fundamentals of C++ through short lessons, interactive (client-side) code walkthroughs, small challenges, and self-check quizzes. It guides absolute beginners from “Hello, World!” to variables, control flow, functions, and classes.
  2. Intended Users: My Client.
  3. Overview of the Website Content: Curriculum overview; lesson pages with examples; a Playground that shows how C++ code works using step-through animations and precomputed outputs, and, quizzes with instant feedback.

Wireframe & Site Map

Page Design Details

Home (index.html)

  1. Purpose: Welcome, explain how to use the site, and direct users to the Curriculum/Lessons.
  2. Audience: All visitors.
  3. Content: Hero with “Start Learning C++”; quick links to Curriculum, Lessons, Playground, Quizzes.
  4. Enter Data? (Y/N): No.
  5. Validation? (Y/N): No.
  6. Buttons/Links/Dropdowns: Buttons & anchors.
  7. Actions: Smooth-scroll; navigate to Curriculum.
  8. Notes: Keep lightweight for fast loading.

Curriculum

  1. Purpose: Show the learning path (Intro → Variables → I/O → Conditionals → Loops → Functions → Arrays/Vectors → Classes).
  2. Audience: Learners, tutors.
  3. Content: Syllabus cards with time estimates and prerequisites.
  4. Enter Data? (Y/N): No.
  5. Validation? (Y/N): No.
  6. Buttons/Links/Dropdowns: Links to each lesson section.
  7. Actions: AJAX reads data/curriculum.json to render the list.
  8. Notes: Client-side only.

Lessons

  1. Purpose: Deliver C++ lessons with step-through example walkthroughs.
  2. Audience: Learners.
  3. Content: Sections for each topic with code snippets, explanations, and outputs.
  4. Enter Data? (Y/N): No (except small controls for demos).
  5. Validation? (Y/N): No.
  6. Buttons/Links/Dropdowns: Tabs/Accordion to switch topics.
  7. Actions: jQuery-UI Tabs or Accordion; examples pulled from data/snippets.json via AJAX; “Step” button animates highlighted lines and reveals precomputed output.
  8. Notes: We do not compile C++ in the browser; outputs are precomputed and explained.

Playground

  1. Purpose: Let learners paste small C++ snippets and see an illustrated run (line-by-line explanation) and typical output (preloaded).
  2. Audience: Learners.
  3. Content: Textarea for code; preset templates (Hello World, loops, functions); output panel.
  4. Enter Data? (Y/N): Yes (code input).
  5. Validation? (Y/N): Basic checks (non-empty, max length).
  6. Buttons/Links/Dropdowns: Run/Reset buttons; template dropdown.
  7. Actions: Client-side parser highlights tokens; simulated output shown for known patterns or show “Explanation only” if unknown.
  8. Notes: No real compilation; no uploads; nothing is stored.

Quizzes

  1. Purpose: Self-check understanding with MCQ/True-False.
  2. Audience: Learners.
  3. Content: Questions on syntax, types, control flow, functions, arrays, classes.
  4. Enter Data? (Y/N): Yes (answers).
  5. Validation? (Y/N): Require a choice before scoring.
  6. Buttons/Links/Dropdowns: Radio buttons; “Check Answers”.
  7. Actions: AJAX loads data/quizzes.json; client-side scoring; show correct/incorrect with explanations.
  8. Notes: Results are not saved (read-only experience).

Resources

  1. Purpose: Provide curated C++ references and starter files.
  2. Audience: Learners & tutors.
  3. Content: Links to C++ references, style guides, and practice sites; downloadable templates.
  4. Enter Data? (Y/N): No.
  5. Validation? (Y/N): No.
  6. Buttons/Links/Dropdowns: Filters by topic; external links.
  7. Actions: AJAX reads data/resources.json; open external links in new tab.
  8. Notes: Use rel="noopener" for security.

Contact

  1. Purpose: Let learners ask questions or request clarifications.
  2. Audience: Learners.
  3. Content: Form (name, email, topic, message) and office hours note.
  4. Enter Data? (Y/N): Yes.
  5. Validation? (Y/N): Yes (HTML5 patterns + JS).
  6. Buttons/Links/Dropdowns: Inputs, select, submit.
  7. Actions: Send via mail handler (Formspree/EmailJS); show success/fail notice.
  8. Notes: No PII stored on the site.

Dynamic Functionality (JavaScript / jQuery / AJAX)

  1. Lesson Tabs & Accordion (Lessons): jQuery-UI Tabs/Accordion to switch topics and keep pages compact.
  2. Code Walkthrough (Lessons & Playground): Step through example C++ snippets; highlight lines and show prepared outputs/explanations loaded from data/snippets.json.
  3. Quizzes (Quizzes): Load questions from data/quizzes.json; instant client-side scoring and feedback.
  4. Curriculum/Resources via AJAX: Render lists from data/curriculum.json and data/resources.json so content is editable without touching HTML.
  5. Shared Header/Footer (All pages): Inject components/header.html and components/footer.html to ensure consistent navigation and validator links.
  6. Interactive lessons / code walkthroughs: Datacamp – Learn || Codecademy
